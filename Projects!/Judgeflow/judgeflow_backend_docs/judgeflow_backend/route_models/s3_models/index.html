§S3/Spaces Configuration Models

This module provides structures and functionality for working with S3-compatible object storage, specifically DigitalOcean Spaces. It handles configuration and authentication for interacting with the storage service.

§Key Components

  • S3Config: Holds configuration for S3/Spaces client
  • from_env(): Factory method to create config from environment variables

§Environment Variables

  • DO_SPACES_BUCKET: The name of the bucket to use
  • DO_SPACES_REGION: The region of the Spaces instance (default: “nyc3”)
  • DO_SPACES_ENDPOINT: The endpoint URL for the Spaces API
  • DO_SPACES_KEY: The access key ID for authentication
  • DO_SPACES_SECRET: The secret access key for authentication

§Example

use judgeflow_backend::route_models::s3_models::S3Config;

// Set up environment variables
std::env::set_var("DO_SPACES_BUCKET", "my-bucket");
std::env::set_var("DO_SPACES_REGION", "nyc3");
std::env::set_var("DO_SPACES_ENDPOINT", "https://nyc3.digitaloceanspaces.com");
std::env::set_var("DO_SPACES_KEY", "my-access-key");
std::env::set_var("DO_SPACES_SECRET", "my-secret-key");

// Create config from environment
let config = S3Config::from_env();
println!("Using bucket: {}", config.bucket);
