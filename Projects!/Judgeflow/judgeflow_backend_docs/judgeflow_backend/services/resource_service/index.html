§Resource Service

This module provides a high-level interface for interacting with Supabase database resources. It handles CRUD (Create, Read, Update, Delete) operations for various resources like hackathons, screenings, and submissions, with built-in validation.

§Features

  • CRUD operations for database resources
  • Built-in request validation
  • Environment-based configuration
  • Error handling and status code mapping

§Configuration

The service requires the following environment variables:

  • SUPABASE_URL: The URL of your Supabase instance
  • SUPABASE_SERVICE_KEY: The service role key for Supabase

§Example

use uuid::Uuid;
use serde_json::json;
use judgeflow_backend::services::resource_service::ResourceService;

let service = ResourceService::new();
let user_id = Uuid::new_v4();

// Create a new item
let new_item = json!({ "name": "Example", "user_id": user_id });
let created = service.create_item("resources", new_item).await?;
println!("Created item: {:?}", created);
