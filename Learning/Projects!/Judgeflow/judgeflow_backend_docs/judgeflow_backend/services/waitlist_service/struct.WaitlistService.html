Expand description

Service for managing the waitlist and interacting with Supabase.

This service provides methods to add users to the waitlist and perform related operations. It handles validation of waitlist entries and communication with the Supabase backend.

§Fields

  • supabase_url: Base URL of the Supabase instance
  • supabase_key: API key for authenticating with Supabase

§Example

use judgeflow_backend::services::waitlist_service::WaitlistService;
use judgeflow_backend::data_models::waitlist_models::WaitlistEntry;

let waitlist_service = WaitlistService::new();
let entry = WaitlistEntry {
    email: "user@example.com".to_string(),
};

match waitlist_service.add_to_waitlist(entry).await {
    Ok(response) => println!("Success: {}", response.message),
    Err(e) => eprintln!("Error: {}", e),
}

pub async fn add_to_waitlist( &self, entry: WaitlistEntry, ) -> Result<WaitlistResponse, String>

Adds a new email address to the waitlist.

This method validates the provided email address and, if valid, adds it to the waitlist in Supabase. The email is validated using the validator crate.

§Arguments
  • entry - A WaitlistEntry containing the email address to add
§Returns
  • Result<WaitlistResponse, String> - A success response if the email was added, or an error message if the operation failed
§Errors

Returns an error if:

  • The email address is invalid
  • The request to Supabase fails
  • The response from Supabase indicates an error
§Example
let service = WaitlistService::new();
let entry = WaitlistEntry {
    email: "user@example.com".to_string(),
};

match service.add_to_waitlist(entry).await {
    Ok(response) => println!("Success: {}", response.message),
    Err(e) => eprintln!("Error: {}", e),
}

pub fn error_to_status_code(_error: &str) -> StatusCode

Maps service error messages to appropriate HTTP status codes.

This utility method converts error strings returned by service methods into the appropriate HTTP status codes for API responses.

§Arguments
  • error - The error message to map to an HTTP status code
§Returns
  • StatusCode - The corresponding HTTP status code
§Mapping Rules
  • All errors currently map to INTERNAL_SERVER_ERROR (500) (This can be extended to handle different error types)
