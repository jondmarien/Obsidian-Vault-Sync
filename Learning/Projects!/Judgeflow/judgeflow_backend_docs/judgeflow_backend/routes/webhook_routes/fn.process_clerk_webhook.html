Expand description

Processes incoming Clerk webhook events for user management.

§Endpoint

POST /api/webhooks/clerk

§Authentication

Verifies request using Svix signatures with CLERK_WEBHOOK_SIGNING_SECRET

§Request Headers

  • svix-id: Unique message ID
  • svix-timestamp: Timestamp of the message
  • svix-signature: Signature for verification

§Request Body

  • ClerkWebhookPayload: The webhook event data

§Returns

  • 200 OK on successful processing
  • 400 Bad Request for invalid signatures
  • 500 Internal Server Error for processing failures

§Example Event Types

  • user.created: New user signup
  • user.updated: User profile update
  • user.deleted: User account deletion

§Note

Signature verification is currently disabled in the code
