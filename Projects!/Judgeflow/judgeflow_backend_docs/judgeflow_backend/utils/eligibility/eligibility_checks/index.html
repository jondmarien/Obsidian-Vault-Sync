§Eligibility Checks Module

This module provides a high-level interface for performing eligibility checks on hackathon participants using the Perplexity API. It includes pre-defined checks for common eligibility criteria and utilities for creating custom checks.

§Features

  • Pre-defined checks for common eligibility criteria (age, location, employment status, etc.)
  • Support for custom eligibility checks
  • Async/await compatible for network operations
  • Comprehensive error handling
  • Built-in logging for audit trails

§Quick Start

use judgeflow_backend::data_models::eligibility_models::EligibilityCheckInput;
use judgeflow_backend::utils::eligibility::eligibility_checks::{
    check_minimum_age, check_eligible_country, perform_all_checks, EligibilityConfig
};
use std::collections::HashMap;

// Create a participant to check
let input = EligibilityCheckInput {
    participant_name: "John Doe".to_string(),
    participant_email: "john.doe@example.com".to_string(),
    participant_age: Some(25),
    participant_country: Some("United States".to_string()),
    participant_employment_status: Some("Student".to_string()),
    additional_data: HashMap::new(),
};

// Perform individual checks
let age_check = check_minimum_age(&input, 18).await?;
let country_check = check_eligible_country(&input, &["United States", "Canada"]).await?;

// Or use the all-in-one function
let config = EligibilityConfig::default();
let all_results = perform_all_checks(&input, &config).await?;

§Custom Checks

You can create custom eligibility checks using the check_criterion function:

let result = check_criterion(
    "custom_check",
    &input,
    Some("Additional context for the check")
).await?;
