Expand description

Service for handling webhook events and external service integration.

This service provides methods to process webhook events from Clerk (authentication service) and interact with Supabase (database) and Discord (notifications).

§Fields

  • supabase_url: Base URL of the Supabase instance
  • supabase_key: API key for authenticating with Supabase

§Example

use judgeflow_backend::services::webhook_service::WebhookService;

let webhook_service = WebhookService::new();
// Use the service to process webhook events...

Fields§

§supabase_url: String

The URL of the Supabase instance (e.g., “https://your-project.supabase.co”)

§supabase_key: String

The service key for the Supabase instance

This key should have appropriate permissions for the operations being performed (e.g., inserting/updating user records)

pub async fn create_user( &self, data: UserCreatedData, ) -> Result<WebhookResponse, String>

Handles a Clerk ‘user created’ webhook event by creating a new user in Supabase.

This method processes a user creation event from Clerk, extracts the relevant user information, and creates a corresponding user record in Supabase.

§Arguments
  • data - The UserCreatedData containing the user information from Clerk
§Returns
  • Result<WebhookResponse, String> - A result containing either a success response with a success message or an error message if the operation fails
§Errors

Returns an error if:

  • The user data is missing an email address
  • The request to Supabase fails
  • The response from Supabase indicates an error
§Example
let service = WebhookService::new();
let user_data = UserCreatedData {
    id: "user_123".to_string(),
    email_addresses: vec![EmailAddress {
        email_address: "user@example.com".to_string(),
        // ... other fields ...
    }],
    first_name: Some("John".to_string()),
    last_name: Some("Doe".to_string()),
    username: Some("johndoe".to_string()),
    // ... other fields ...
};

match service.create_user(user_data).await {
    Ok(response) => println!("User created: {}", response.message),
    Err(e) => eprintln!("Failed to create user: {}", e),
}

pub async fn process_clerk_webhook( &self, event: UserCreatedData, ) -> Result<WebhookResponse, String>

Processes an incoming Clerk webhook event for user creation.

This is a lower-level method that handles the actual creation of a user in Supabase based on data from a Clerk webhook event. It validates the input data, constructs the user record, and sends it to Supabase.

§Arguments
  • event - The UserCreatedData from the Clerk webhook
§Returns
  • Result<WebhookResponse, String> - A result indicating success or failure
§Errors

Returns an error if:

  • The email address is missing or invalid
  • The request to Supabase fails
  • The response from Supabase indicates an error
§Note

This method is similar to create_user but is specifically designed to be called from webhook handlers. It includes additional validation and error handling specific to webhook processing.

§Example
let service = WebhookService::new();
let event_data = UserCreatedData {
    id: "user_123".to_string(),
    email_addresses: vec![
        // Clerk email address structure...
    ],
    first_name: Some("John".to_string()),
    last_name: Some("Doe".to_string()),
    // ... other fields ...
};

match service.process_clerk_webhook(event_data).await {
    Ok(response) => println!("Webhook processed: {}", response.message),
    Err(e) => eprintln!("Webhook processing failed: {}", e),
}
