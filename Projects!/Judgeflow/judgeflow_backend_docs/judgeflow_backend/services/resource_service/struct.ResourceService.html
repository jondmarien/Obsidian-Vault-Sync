pub struct ResourceService {
    supabase_url: String,
    service_key: String,
}
Expand description

A client for interacting with Supabase database resources.

This struct provides methods to perform CRUD operations on various resources in a Supabase database. It handles authentication, request building, and response parsing.

§Fields

  • supabase_url: The base URL of the Supabase instance
  • service_key: The service role key for authenticating with Supabase

§Panics

The new function will panic if the required environment variables are not set.

pub async fn get_items( &self, resource: &str, user_id: &Uuid, ) -> Result<Value, String>

Retrieves all items from a specified resource table filtered by user ID.

§Arguments
  • resource - The name of the resource table to query (e.g., “hackathons”, “submissions”)
  • user_id - The UUID of the user to filter results by
§Returns
  • Ok(Value) - A JSON value containing the retrieved items on success
  • Err(String) - An error message if the request fails
§Errors

Returns an error if:

  • The HTTP request fails
  • The response cannot be parsed as JSON
  • The Supabase API returns a non-success status code
§Example
let service = ResourceService::new();
let user_id = Uuid::parse_str("123e4567-e89b-12d3-a456-426614174000")?;
 
match service.get_items("hackathons", &user_id).await {
    Ok(items) => println!("Found hackathons: {:?}", items),
    Err(e) => eprintln!("Error retrieving hackathons: {}", e),
}

pub async fn update_item( &self, resource: &str, id: &str, user_id: &Uuid, body: Value, ) -> Result<Value, String>

Updates an existing item in the specified resource table.

§Arguments
  • resource - The name of the resource table (e.g., “hackathons”, “submissions”)
  • id - The ID of the item to update
  • user_id - The UUID of the user who owns the item (for authorization)
  • body - A JSON value containing the fields to update
§Returns
  • Ok(Value) - The updated item as a JSON value on success
  • Err(String) - An error message if the update fails
§Errors

Returns an error if:

  • The HTTP request fails
  • The response cannot be parsed as JSON
  • The Supabase API returns a non-success status code
  • The user is not authorized to update the item
§Example
let service = ResourceService::new();
let user_id = Uuid::parse_str("123e4567-e89b-12d3-a456-426614174000")?;
let item_id = "abc123";
 
let updates = json!({
    "description": "Updated description for the hackathon"
});

match service.update_item("hackathons", item_id, &user_id, updates).await {
    Ok(updated) => println!("Updated: {:?}", updated),
    Err(e) => eprintln!("Error updating item: {}", e),
}

pub fn validate_resource( &self, resource: &str, body: &Value, ) -> Result<(), String>

Validates resource data against the appropriate model.

This is an internal method used to validate data before it’s sent to the database. It ensures that the data conforms to the expected schema for the resource.

§Arguments
  • resource - The name of the resource being validated
  • body - The JSON data to validate
§Returns
  • Ok(()) - If the data is valid
  • Err(String) - A validation error message if the data is invalid
§Note

Currently supports validation for:

  • “hackathon” - Validates against the Hackathon model
  • “screenings” - Validates against the Screening model
  • “submissions” - Validates against the Submission model Other resource types will pass validation without checking.
