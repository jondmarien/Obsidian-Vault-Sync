Function split_csv_record

fn split_csv_record(record: &str) -> Vec<String>
Expand description

Parses a single CSV line into individual fields, handling quoted values and escaped quotes.

This is a robust CSV field parser that can handle:

  • Quoted fields containing commas
  • Escaped quotes (“” inside quotes becomes “)
  • Fields spanning multiple lines
  • Trims whitespace from unquoted fields
  • Empty fields

§Arguments

  • record - A single line from a CSV file (may be part of a multi-line record)

§Returns

A vector of strings, where each string is a field from the CSV record

§Example

use judgeflow_backend::utils::csv_processing::fix_csv::split_csv_record;

let line = r#"1,"John, Doe","123 Main St"""#;
let fields = split_csv_record(line);
assert_eq!(fields, vec!["1", "John, Doe", "123\""]);

§Implementation Notes

  • Handles both quoted and unquoted fields
  • Properly handles escaped quotes within quoted fields
  • Trims whitespace from unquoted fields
  • Returns empty strings for empty fields (,,)
