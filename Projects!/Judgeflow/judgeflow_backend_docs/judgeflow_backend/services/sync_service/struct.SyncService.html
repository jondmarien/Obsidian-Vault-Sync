Expand description

Service for synchronizing user data between Clerk and Supabase.

This service handles the entire synchronization process, including:

  1. Fetching users from Clerk’s API with pagination
  2. Transforming the user data into the application’s format
  3. Upserting the data into Supabase

§Fields

  • clerk_secret: API key for authenticating with Clerk
  • supabase_url: Base URL of the Supabase instance
  • supabase_key: API key for authenticating with Supabase
  • client: Reusable HTTP client for making requests

§Example

use judgeflow_backend::services::sync_service::SyncService;

let sync_service = SyncService::new()?;
let result = sync_service.sync_all_users().await?;
println!("Synchronization result: {}", result.message);

Fetches all users from Clerk’s API with pagination support.

async fn sync_user_to_supabase( &self, user_data: &UserData, ) -> Result<(), String>

Synchronizes a single user’s data to Supabase.

This method performs an upsert operation on the Supabase user table, using the email address as the conflict resolution key. If a user with the same email already exists, their record will be updated; otherwise, a new record will be created.

§Arguments
  • user_data - A reference to the UserData to sync
§Returns
  • Result<(), String> - Ok(()) on success, or an error message on failure
§Errors

Returns an error if:

  • The request to Supabase fails
  • The response indicates an error status
§Example
let sync_service = SyncService::new()?;
let user_data = UserData {
    email: "user@example.com".to_string(),
    full_name: Some("John Doe".to_string()),
    role: "participant".to_string(),
};
sync_service.sync_user_to_supabase(&user_data).await?;

pub async fn sync_all_users(&self) -> Result<SyncResponse, String>

Synchronizes all users from Clerk to Supabase.

This method performs a full synchronization by:

  1. Fetching all users from Clerk
  2. Transforming each user to the application’s format
  3. Upserting each user into Supabase
§Returns
  • Result<SyncResponse, String> - A summary of the synchronization operation or an error message
§Errors

Returns an error if:

  • Unable to fetch users from Clerk
  • Any individual user fails to sync (though the operation will continue with other users)
§Example
let sync_service = SyncService::new()?;
let result = sync_service.sync_all_users().await?;
println!("Synchronized {} users", result.count.unwrap_or(0));
