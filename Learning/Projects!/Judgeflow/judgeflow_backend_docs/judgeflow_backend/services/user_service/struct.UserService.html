Expand description

Service for managing user data and authentication.

This service provides methods to interact with user data stored in Supabase, including retrieving user information and converting between different user models.

§Fields

  • supabase_url: Base URL of the Supabase instance
  • supabase_key: API key for authenticating with Supabase

§Example

use judgeflow_backend::services::user_service::UserService;

let user_service = UserService::new();
let user = user_service.get_user_by_clerkid("clerk_123").await?;
println!("User email: {}", user.email);

Fields§

§supabase_url: String

The URL of the Supabase instance (e.g., “https://your-project.supabase.co”)

§supabase_key: String

The service key for authenticating with Supabase

This key should have appropriate permissions for the operations being performed (e.g., reading user data)

pub async fn get_user_by_clerkid( &self, clerk_id: &str, ) -> Result<User, (StatusCode, String)>

Retrieves a user by their Clerk ID from the database.

This method queries the Supabase database for a user with the specified Clerk ID and returns the corresponding user data if found.

§Arguments
  • clerk_id - The Clerk ID of the user to retrieve
§Returns
  • Result<User, (StatusCode, String)> - The user data if found, or an error tuple containing an HTTP status code and error message
§Errors

Returns an error tuple if:

  • The request to Supabase fails
  • The user is not found
  • The response cannot be parsed
§Example
let user_service = UserService::new();
match user_service.get_user_by_clerkid("clerk_123").await {
    Ok(user) => println!("Found user: {}", user.email),
    Err((status, msg)) => eprintln!("Error ({}): {}", status, msg),
}
