§Sonar-PPLX Integration Module

This module provides integration with the Perplexity AI’s Sonar model for natural language processing tasks, particularly focused on eligibility checking for hackathon participants.

§Features

  • Generic eligibility checking using LLM-powered semantic analysis
  • Predefined checks for common eligibility criteria (geography, age, employment, etc.)
  • Extensible architecture for adding new check types
  • Structured response parsing with confidence scoring

§Quick Start

use judgeflow_backend::data_models::eligibility_models::EligibilityCheckInput;
use judgeflow_backend::utils::sonar_pplx::{SonarPplx, SonarPplxTrait};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let input = EligibilityCheckInput {
        participant_name: "John Doe".to_string(),
        participant_email: "john@example.com".to_string(),
        participant_age: Some(22),
        participant_country: Some("Canada".to_string()),
        participant_employment_status: Some("Student".to_string()),
        additional_data: std::collections::HashMap::new(),
    };

    let sonar = SonarPplx;
    let result = sonar.generic_check("minimum_age", &input, Some("Minimum age is 18")).await?;
     
    println!("Eligibility check result: {:?}", result);
    Ok(())
}

§Architecture

  • SonarPplxTrait: Defines the interface for eligibility checking
  • SonarPplx: Implements the trait for making actual API calls
  • Predefined check functions: check_geography, check_age, etc.
  • generic_check: A flexible function for custom eligibility criteria
