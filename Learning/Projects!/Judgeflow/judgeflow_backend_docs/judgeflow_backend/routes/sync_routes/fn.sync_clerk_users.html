pub async fn sync_clerk_users() -> impl IntoResponse
Expand description

Synchronizes all users from Clerk to the application database.

§Endpoint

POST /api/sync-users

§Description

This endpoint triggers a full synchronization of user data from Clerk (authentication provider) to the application’s database. It’s typically used during initial setup or when manual synchronization is needed.

§Authentication

Requires admin privileges.

§Returns

  • 200 OK with sync results on success
  • 500 Internal Server Error if synchronization fails

§Example Response (Success)

{
  "success": true,
  "message": "Successfully synced 42 users",
  "count": 42
}

§Example Response (Error)

{
  "success": false,
  "message": "Failed to fetch users from Clerk: Connection timeout",
  "count": null
}
